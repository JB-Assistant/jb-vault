## Summary

Alex Finn shares lessons from 300+ hours of OpenClaw usage. This is a comprehensive beginner-to-advanced tutorial covering setup, models, use cases, multi-agent workflows, security, and more.

**Key topics covered:**
1. **What OpenClaw is** — fully autonomous, open-source AI agent that lives on your device
2. **Hosting** — local device strongly recommended over VPS; Mac Mini or any old laptop works
3. **Models (Brain & Muscles)** — Opus 4.7 as orchestrator, ChatGPT 5.4 for coding, cheap models (Gemini) for research/writing, local models (GLM 5.1, Qwen 3.6) for free 24/7 tasks
4. **Communication** — Telegram best for chatting (multi-topic groups for context), Discord for automated workflows
5. **Finding use cases** — Brain dump everything about yourself → list daily manual tasks → ask OpenClaw to suggest personalized workflows
6. **Example use cases** — Linear integration for vibe coding, content scouting + script drafting, stock research, prototyping on the go
7. **OpenClaw vs Hermes** — Hermes slightly faster/more token-efficient, OpenClaw has better memory; best to run both side-by-side as insurance
8. **Multi-agent approach** — run 2 agents so one can fix the other when updates break things
9. **Mission Control** — custom software platform where OpenClaw builds its own tools as needed
10. **Security** — no catastrophic incidents reported; use good judgment with prompts, OpenClaw only does what you tell it

## Key Takeaways

1. **Never use VPS** — local devices are more secure, performant, and powerful; zero advantages to VPS hosting
2. **Opus 4.7 is king for orchestration** — nothing else comes close for reliability in task completion ("will crawl to the finish line with a severed leg")
3. **ChatGPT 5.4 for coding muscle** — best balance of cost, limits, and quality
4. **Local models unlock free 24/7 operation** — GLM 5.1, Qwen 3.6 can scrape/research continuously at zero API cost
5. **Telegram > Discord for daily use** — multi-topic groups save tokens by keeping context separate
6. **Brain dump + task tracking → personalized workflows** — more valuable than copying others' use cases
7. **Run both OpenClaw + Hermes** — insurance against breakage, plus you discover which fits your workflow better
8. **Mission Control = your custom toolbox** — build tools on demand, don't copy someone else's
9. **Security is about prompt judgment** — not infrastructure; OpenClaw only does what you tell it
10. **Scale slowly** — old laptop → Mac Mini → Mac Studio, don't drop $20K upfront

## Hermes Angle

This video directly mentions Hermes as an alternative/complement to OpenClaw. Alex's take:
- Hermes is **slightly more performant and token-efficient**
- OpenClaw has **better memory/context retention**
- He recommends running **both side-by-side** for redundancy
- When one breaks after an update, the other can fix it
- He frames them as "basketball teams" — tribalism is pointless, use what works

This is a useful reference for understanding how the OpenClaw community perceives Hermes and the multi-agent landscape.
