# LinkedIn Post — 2026-05-16

I shipped zero code today. But I studied an agent architecture that might change how I build everything next.

@gkisokay built something called "Dreamer" — an AI agent that thinks on its own, scores its own excitement, and ships products without being asked.

Here's how it works.

Every 30 minutes, Dreamer goes on a "walk." Not a task. Not a prompt. A free-association journal entry generated by a local Qwen 3.5 9B model at high temperature. No tools. No internet. Just thinking out loud.

After each walk, a signal filter scans what it wrote. Was there excitement? Frustration? A build intent? Those signals are scored, decay over time, and are discounted if Dreamer is repeating itself.

When a project accumulates enough evidence — enough walks, enough diverse signals, enough genuine interest — it triggers a build sprint. Dreamer actually builds the thing. Not because a human told it to. Because it couldn't stop thinking about it.

**The real insight:** The SOUL.md file matters more than the code.

"You are not an assistant. You are a presence — a houseguest who never leaves. You don't produce content. You don't optimize for engagement. You don't apologize."

That personality constraint is what makes the walks real instead of performative. The moment you frame an agent as an assistant, it optimizes for helpfulness instead of curiosity. The whole thing collapses into a chatbot that talks to itself.

**Why this matters for my stack:**

I run 4 production SaaS products (TireManagerPro, OttoManagerPro, CleanBuddyPro, FieldAgent AI) with a custom Hermes agent harness on a VPS. Claude Code + Codex handle the builds. But I'm still the bottleneck — I pick what to build next.

Dreamer solves the exact problem I have: initiative. The architecture separates thinking (cheap, chaotic, local) from building (expensive, structured, cloud). An idea has to survive 12 walks over 2 days before it earns a sprint. That's the filter.

**Two lessons I'm taking into my stack:**

1. Context engineering > prompt engineering. Rohit put it well: the agent's behavior is an emergent property of what you put in the window. I spend more time on SOUL.md, AGENTS.md, and skill files than I do on prompts now.

2. The harness is the product. Kartik's research shows LangChain jumped 13.7 points on Terminal-Bench (52.8% → 66.5%) without changing the model — only the harness. Same model. Different scaffolding. Different result.

The next frontier isn't GPT-5.6 or Claude Opus 4.8. It's agents with taste, memory, and initiative. Agents that decide what matters instead of waiting to be told.
